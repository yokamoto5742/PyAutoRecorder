# PyAutoRecorder

Windows上でマウスとキーボード操作を自動記録・再生するGUIツール。条件付き実行、タイマースケジューリング、グローバルホットキー制御を備えており、複雑なマクロ処理をコード不要で構築できます。

## 特徴

- **グローバル記録**: マウスとキーボード操作をアプリケーション全体から自動感知・記録
- **3フェーズマクロ**: 初期処理 → 繰り返し処理 → 最後の処理 の構成で複雑なワークフローをサポート
- **条件付き実行**: ウィンドウ表示検知、クリップボード内容、画面上の色判定、ファイル存在確認など14種類の条件判定機能
- **タイマースケジューリング**: 特定時刻または経過時間後に自動再生・停止
- **クリップボード変数**: マクロ再生時にクリップボード値をもとに {VAR:キー名} を置換
- **IME自動制御**: Windows IMEの自動切り替え対応
- **グローバルホットキー**: デフォルト F10（再生/停止）、F11（一時停止）で即座に操作
- **トレイランチャー**: 頻繁に使用するマクロをシステムトレイから直接実行
- **要素ハイライト**: 要素ピッカー実行時に、カーソル下のUIコントロールを赤枠で表示
- **マルチプロセス対応**: 子機による手動マウス記録、キーボード操作の独立記録

## インストール方法

### 前提条件
- **OS**: Windows 10/11
- **Python**: 3.13以上
- **パッケージ管理**: `uv`（未インストール時は [uv公式ドキュメント](https://docs.astral.sh/uv/getting-started/) を参照）

### セットアップ手順

1. **リポジトリをクローン**
   ```bash
   git clone https://github.com/yokamoto5742/PyAutoRecorder
   cd PyAutoRecorder
   ```

2. **依存パッケージをインストール**
   ```bash
   uv sync
   ```

3. **設定ファイルの確認**
   - `utils/config.ini` の内容を確認（ホットキー、ログレベルなど）
   - 必要に応じて設定を編集

4. **アプリケーション起動確認**
   ```bash
   python main.py
   ```
   メインウィンドウが表示されればインストール成功

5. **（オプション）スタンドアロン実行ファイルをビルド**
   ```bash
   python build.py
   ```
   `dist/PyAutoRecorder/` フォルダに実行ファイルが生成されます

## 使い方

### 基本的なマクロ作成フロー

1. **アプリケーション起動**
   ```bash
   python main.py
   ```

2. **操作を記録**
   - ツールバーの「記録」ボタン → 「自動記録」を選択
   - マウス・キーボード操作を実行（画面右下の停止ボタンで終了）
   - または手動で個別操作を追加

3. **操作内容を確認・編集**
   - メインウィンドウのリストでマウスクリック、キーボード操作、クリック方法を確認
   - 必要に応じて各項目をダブルクリックで編集

4. **ファイルに保存**
   ```
   「新規」→ ファイル名を入力 → 「保存」
   ```
   （ファイル形式: `*.par`）

5. **マクロを再生**
   - ツールバーの「再生」ボタンをクリック
   - または設定したホットキー（デフォルト F10）で再生開始

### 記録方法（3種類）

#### 自動記録
- ツールバー「記録」→ 「自動記録」
- 全マウス・キーボード操作を自動感知
- 画面右下の停止ボタンで終了

#### 手動マウス操作の記録（子機）
- ツールバー「記録」→ 「手動 - マウス操作の記録（子機）」
- 子機ウィンドウが表示される
- 子機上で操作内容を選択して記録（クリック方法を指定可能）

#### 手動キーボード操作の記録
- ツールバー「記録」→ 「手動 - キーボード操作の記録」
- キー入力表記（例：`^a`=Ctrl+A、`{ENTER}`=Enter キー）
- トークン形式で直接入力

### クリップボード変数の利用

マクロ再生時にクリップボード内の値をテキスト送信に使用できます：

1. **キーボード操作欄に変数を指定** 
   - キー入力欄に `{VAR:キー名}` と記述（例：`{VAR:ID}`、`{VAR:PASSWORD}`）
   - 項目編集ダイアログのキーボード操作欄には「変数」ボタンで `{VAR:}` を挿入

2. **再生時の確認フロー**
   - マクロ再生開始時にクリップボードを読み込み
   - 指定されたキー（タブ区切り・複数行対応）の値を確認ダイアログで表示
   - ユーザーが確認後、「OK」で変数を取り込み再生開始

3. **クリップボードフォーマット例**
   - Excelからのコピー（タブ区切り）: `ID\t001\nPASSWORD\tabc123`
   - テキストファイル（1行＝1項目）

### 条件判定の設定

項目を編集する際に「条件判定」タブで以下が設定可能：

| 条件タイプ | 例 |
|-----------|-----|
| ウィンドウ表示検知 | タイトルが「Notepad」のウィンドウが表示されるまで待機 |
| クリップボード検知 | クリップボード内容に特定の文字列を含むかチェック |
| 色判定 | 指定座標の画面色が一致するまで待機 |
| ファイル判定 | ファイルが存在・生成されるのを待機 |
| 日時判定 | 特定の時刻を迎えるまで待機 |
| 繰り返し数判定 | N回目の繰り返しのみ実行 |

詳細は app/constants.py の `CONDITION_LABELS` を参照。

### ホットキー

| キー | 動作 |
|-----|------|
| F10 | 再生/停止（トグル） |
| F11 | 一時停止/再開（トグル） |

カスタマイズ方法：`utils/config.ini` の `[Hotkeys]` セクションを編集

```ini
[Hotkeys]
play_stop = <f10>      # pynput記法（例: <ctrl>+<alt>+p）
pause = <f11>
```

### `.par` ファイルをダブルクリックして再生

エクスプローラー上で `.par` ファイルをダブルクリックするだけで、アプリが自動的に起動して即座に再生されます（起動済みの場合はメインウィンドウを増やさず、既存インスタンス側で再生されます）。利用するには事前に一度だけ、Windows側でファイルの関連付けを行ってください。

1. 任意の `.par` ファイルを右クリック
2. 「プログラムから開く」→「別のプログラムを選択」を選択
3. `PyAutoRecorder.exe`（`python build.py` でビルドした場合は `dist/PyAutoRecorder/PyAutoRecorder.exe`）を選択
4. 「常にこのアプリを使って `.par` ファイルを開く」にチェックを入れて OK

以降は `.par` ファイルをダブルクリックするだけで、メインウィンドウを表示せずに再生され、再生完了後は自動的にアプリが終了します。

### タイマースケジューリング

ツールバーの「タイマー」ボタン→ ダイアログで設定

- **再生タイマー**: 指定時刻または経過時間後に自動再生
- **停止タイマー**: 指定時刻または経過時間後に自動停止
  - 「すべて停止」: 繰り返し処理を中断
  - 「最後の処理をする」: 繰り返しを終了後、最後の処理を実行

## 主な機能

### 入力記録（service/recorder.py）

- **マウス移動**: 10px 以上の移動をドラッグとして記録
- **クリック判定**: 0.4秒・5px 以内の2クリックをダブルクリックに統合
- **キーボード入力**: 連続キー入力を1項目にトークン化（{ENTER}、^A など）
- **グローバルフック**: pynput で全アプリケーションの入力をリアルタイム感知

### 再生エンジン（service/player.py）

```python
# 再生エンジン（QThread）の基本構造
player = MacroPlayer(macro_file)
player.item_started.connect(on_item_started)  # 項目開始
player.error_occurred.connect(on_error)       # エラー通知
player.playback_finished.connect(on_done)     # 再生終了
player.start()                                # 再生開始
```

実行順序：
1. 最初の処理（repeat_count に関わらず1回）
2. 繰り返し処理（macro.settings.repeat_count 回）
3. 最後の処理（repeat_count に関わらず1回）

### キー入力表記（service/key_notation.py）

| 表記 | 意味 |
|-----|------|
| `a` | 小文字 a |
| `A` | 大文字 A（Shift+a） |
| `^a` | Ctrl+A |
| `+a` | Shift+A |
| `%a` | Alt+A |
| `{ENTER}` | Enter キー |
| `{TAB}` | Tab キー |
| `{F1}` | F1 キー |
| `^{DEL}` | Ctrl+Delete |
| `{VAR:キー名}` | クリップボード変数（再生時に置換） |
| `{WAIT}` | ウェイト（一時停止） |
| `{CLIP}` | クリップボード取得 |
| `{CLEAR}` | 入力フィールドクリア |
| `{IME}` | IME制御 |

### データ永続化（service/models.py）

マクロファイルは JSON 形式で保存：

```json
{
  "initial": [
    { "type": "move", "x": 100, "y": 200 },
    { "type": "left_click", "x": 100, "y": 200 }
  ],
  "loop": [...],
  "final": [...],
  "settings": { "repeat_count": 3 }
}
```

## トラブルシューティング

### 「グローバルフックが動作しない」

**原因**: 管理者権限がない、または他のアプリケーションがキーボード・マウスフックを占有している

**対策**:
- PyAutoRecorder をシステム管理者として実行
- バックグラウンドで動作しているセキュリティソフト（Kaspersky、Avastなど）を一時停止
- IME や他のマクロツール（AutoHotkey など）との競合を確認

### 「IME制御に失敗した」

**原因**: Windows IME が無効化されている、または日本語入力システムが非標準設定

**対策**:
- Windows 設定から日本語入力システム（IME）が有効になっていることを確認
- オプション設定で IME 制御を無効にする（conditions.py の機能を無効化）
- imm32.dll への問題がないか確認（pywin32 が正常にインストールされているか）

### 「マウス座標がズレる」

**原因**: マルチモニター環境での座標計算、または画面スケーリング（125%、150%）

**対策**:
- 画面スケーリングを 100% に設定する
- 記録時と再生時で同じモニター構成を使用
- 子機で手動マウス記録を使用（座標の精度が向上）

### 「キーボード入力が認識されない」

**原因**: キー表記の不正、修飾キー順序の問題

**対策**:
- キー表記が `service/key_notation.py` で定義されたものと一致しているか確認
- 修飾キーは `^` → `+` → `%` の順序で記述（例: `^+a` = Ctrl+Shift+A）
- テスト的に item_editor_dialog で手動記録を試す

### 「ホットキーが効かない」

**原因**: config.ini の記法エラー、または別アプリケーションが同じキーを占有

**対策**:
- `config.ini` の記法を確認（pynput 記法: `<f10>`、`<ctrl>+<alt>+p` など）
- デフォルト（F10、F11）に戻して動作確認
- Task Manager や他のアプリケーションのホットキー設定を確認

### 「再生中に画面操作が反応しない」

**原因**: pyautogui の FAILSAFE（画面左上隅へのマウス移動で緊急停止）が誤動作

**対策**:
- 再生中は画面左上隅にマウスを移動しない
- マクロの最中にキーボード・マウスで手動操作しない
- 必要に応じて player.stop() 関数を呼び出して再生を停止
